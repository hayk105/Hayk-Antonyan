Git add.
    Փոփոխությունները աշխատանքային գրացուցակից տեղափոխում է բեմադրության տարածք: Սա ձեզ հնարավորություն է տալիս պատրաստել ակնթարթ՝ նախքան այն ներկայացնելը պաշտոնական պատմությանը:
Git Branch.
    Այս հրամանը ձեր ընդհանուր նշանակության մասնաճյուղի կառավարման գործիքն է: Այն թույլ է տալիս ստեղծել մեկուսացված զարգացման միջավայրեր մեկ պահեստում:
Git commit.
    Վերցնում է բեմադրված նկարը և հանձնում այն ​​նախագծի պատմությանը: Համակցված git add-ի հետ՝ սա սահմանում է հիմնական աշխատանքային հոսքը Git-ի բոլոր օգտագործողների համար
Git fetch.
    Ներբեռնումը ներբեռնում է branch-ը մեկ այլ պահոցից՝ դրա հետ կապված բոլոր պարտավորությունների և ֆայլերի հետ միասին: Բայց, այն չի փորձում որևէ բան ինտեգրել ձեր տեղական պահեստում: Սա ձեզ հնարավորություն է տալիս ստուգել փոփոխությունները՝ նախքան դրանք ձեր նախագծին միաձուլելը:
Git pull.
    Pulling-ը git fetch-ի ավտոմատացված տարբերակն է: Այն ներբեռնում է մասնաճյուղը հեռավոր պահոցից, այնուհետև այն անմիջապես միացնում է ընթացիկ մասնաճյուղին: Սա svn թարմացման Git համարժեքն է:
